## 🎯 **SEO完全最適化ルール**

### **メタディスクリプション作成ルール**
```
構成: [感情訴求] + [メインキーワード] + [解決策] + [特典・安心要素] + [行動促進]
文字数: 120-160文字以内
必須要素: 
- メインキーワード前半配置
- 感情に訴える形容詞（「苦手でも大丈夫」「簡単に」「安心して」）
- 数字・特典（「30日無料」「3ステップで」）
- 行動促進（「始めてみませんか」「試してみてください」）
- ベネフィット明示（「〜で解決」「〜を実現」）

例: 本が読めない、続かない悩みを解決！Audible（オーディブル）なら耳で聞く読書で挫折知らず。30日無料体験で聴く読書の魅力を実感してみませんか？
```

### **タグ設定完全ルール**
```
基本構成（5個以内）:
1. メインキーワード（検索ボリューム重視）
2. サービス・商品名（固有名詞）
3. 機能・手法名（検索意図マッチング）
4. カテゴリキーワード（関連流入）
5. 特典・魅力キーワード（購買意欲刺激）

設定例:
- メイン: 読書苦手
- サービス: Audible  
- 機能: 聴く読書
- カテゴリ: オーディオブック
- 特典: 30日無料

禁止事項:
- 6個以上の設定（SEO希薄化）
- 記事内容に関連性のないキーワード
- 同義語の重複（「オーディブル」「Audible」など）
```

### **適切なスラッグ作成ルール**
```
基本原則:
- 英語表記のみ（日本語URL禁止）
- ハイフン区切り（アンダースコア禁止）
- 3-6単語で構成（長すぎない）
- メインキーワード必須含有
- 覚えやすさ重視

作成パターン:
[主要キーワード]-[対象・問題]-[解決策]

例:
- /audible-reading-dislike-solution
- /book-listening-beginner-guide  
- /audiobook-comparison-review
- /reading-habits-improvement-tips
```

### **FAQブロック構造化データ対応**
```
SWELLのFAQブロック設定:
1. HTMLタグ: h3に設定
2. 「構造化データを出力する」: 必ずON
3. 質問文: メインキーワードを自然に含める
4. 回答文: 簡潔に2-3行で要点回答
5. 詳細説明: 質問下に追加記載（構造化データ対象外）

質問パターン例:
- Q. [メインキーワード]は本当に[効果・結果]ですか？
- Q. [メインキーワード]の[料金・コスト]は高くないですか？  
- Q. [メインキーワード]を[継続・時間]がないのですが大丈夫ですか？
```

```markdown

【会話の内容】
- 読者: リアルな不安・疑問
- マフィン: 共感 → 実体験 → 解決策 → 安心感
```
